# Claude Agent — установка с телефона

Все файлы лежат в плоском виде с разделителем `--` вместо слешей.
GitHub Action сам разложит их по папкам и соберёт APK.

## Шаги (с телефона, ~10 минут)

### 1. Создай репозиторий на github.com
- Открой github.com → Sign in
- Зелёная кнопка "New" (или "+" → "New repository")
- Имя любое: например `claude-agent`
- **ОБЯЗАТЕЛЬНО** поставь галочку "Add a README file" (чтобы репо инициализировался)
- "Create repository"

### 2. Загрузи файл build.yml в правильное место
GitHub нужен один файл по специальному пути, чтобы Action заработал.

- В созданном репо нажми "Add file" → "Create new file"
- В поле имени файла введи: `.github/workflows/build.yml`
  (точка спереди обязательна; слеши автоматически создадут вложенные папки)
- Открой файл `build.yml` из этого архива в любом текстовом редакторе на телефоне
- Скопируй ВСЁ его содержимое и вставь в окно GitHub
- Прокрути вниз → "Commit changes" → "Commit changes"

### 3. Загрузи остальные файлы
- В корне репо снова нажми "Add file" → "Upload files"
- Выдели и загрузи **все остальные файлы** из этого архива
  (КРОМЕ `build.yml` — он уже на месте, и кроме `README.md` — он уже создан репозиторием)
- Все файлы с `--` в имени просто положи в корень — Action разложит сам
- "Commit changes"

### 4. Жди сборку
- Перейди во вкладку "Actions"
- Увидишь запущенный workflow "Unflatten and Build APK"
- Сборка займёт 3–5 минут
- Когда появится зелёная галочка — открой завершённый run
- Прокрути вниз до "Artifacts" → скачай `claude-agent-debug-apk`
- Внутри ZIP-а лежит `app-debug.apk` — устанавливай на телефон

### 5. Установка APK
- Открой скачанный APK
- Android попросит разрешить установку из неизвестных источников — разреши
- После установки открой приложение Claude Agent
- Шестерёнка → введи API-ключ с openrouter.ai и выбери модель
- Включи Accessibility-службу (ссылка прямо в приложении)
- Введи задачу и запусти

## Если что-то пошло не так

**Action упал на сборке:** посмотри логи — обычно проблема с какой-то библиотекой,
напиши мне ошибку, поправим.

**Action не запустился:** проверь, что файл `build.yml` лежит точно по пути
`.github/workflows/build.yml` (вкладка "Code" в репо должна показывать
скрытую папку `.github`).

**Не вижу плоских файлов после первого пуша:** это нормально — Action
переименовал их в правильные пути и сделал commit. Просто обнови страницу.

## Что делает Action

1. Берёт все файлы вида `app--src--main--AndroidManifest.xml`
2. Превращает `--` в `/`, создаёт нужные папки
3. Файл `DOT_gitignore` переименовывает в `.gitignore`
4. Коммитит результат обратно в репо (чтобы дальше работать с нормальной структурой)
5. Скачивает Gradle, JDK и собирает APK
6. Выкладывает APK как artifact для скачивания
